Continuity change

This application helps you to calculate the Continuity Change algorithm for your datasets. It consists of four steps:

  • Create the application database starting from a csv file

  • Insert the memory value and select which kind of result you want: the application lets you compare each entity year with its own history, with the history of all the other entities and with its own history plus the history of all the other entities

  • Export the result set as a csv output file

  • Drop the dataset and restart

1. Create your dataset

The uploader accepts a text file in csv format (Comma Separated Values). In this format, each row is normally represented by a text line, splitted in fields by an appropriated symbol (a comma in this case); each field represent a value. The file must contain four columns:

  • the entity id
  • the product id
  • the time
  • the attribute

2. Insert memory value and options

In this section you can decide to calculate the algorithm comparing each entity attribute with its own history, with the hisotry of all the other entities, with its own history plus the hisotry of all the other entities. Then, insert the memory value (e.g.: 5), which represents the length of the past history to compare. IMPORTANT: you must insert a memory value to proceed!

3. Calculate results

Click the bottom button to execute the algoritm; the result set will be downloaded as a csv file. Foreach entity will be given the following values, according to the options selected at step two

  • CHANGE1 and DEPTH1 are the result of the comparison with enitity history
  • CHANGE2 and DEPTH2 are the result of the comparison with all the other enitity histories
  • CHANGE3 and DEPTH3 are the result of the comparison with enitity history + all the other enitity histories
Export as CSV

4. Drop database and repeat

Click the bottom button to delete the dataset created at step one and restart with a new iteration.
